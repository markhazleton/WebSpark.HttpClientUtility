WebSpark.HttpClientUtility

Drop-in HttpClient wrapper for .NET 8-10 with Polly resilience, response caching, OpenTelemetry, and batch execution orchestration. One-line setup eliminates boilerplate for retries, circuit breakers, correlation IDs, and structured logging. Perfect for microservices, APIs, and web scrapers. Includes authentication providers (Bearer, Basic, API Key), concurrent requests, fire-and-forget operations, streaming support, and environment x user x request batch runs with progress and statistics. Supports .NET 8 LTS (until Nov 2026), .NET 9, and .NET 10. For web crawling features, see WebSpark.HttpClientUtility.Crawler package.

Features

✅ Simple one-line service registration
No complex configuration required. Just call services.AddHttpClientUtility() and you're ready to go.

✅ Built-in resilience with Polly integration
Automatic retry policies, circuit breakers, and timeout handling out of the box.

✅ Automatic response caching
Intelligent caching with configurable duration and cache invalidation strategies.

✅ Comprehensive telemetry and logging
OpenTelemetry integration with correlation IDs for distributed tracing.

✅ Web crawling capabilities
Robots.txt parsing, SignalR progress updates, and concurrent crawling support.

✅ Flexible authentication
Built-in providers for Bearer tokens (with dynamic refresh), Basic auth, and API keys with factory helpers.

✅ Batch execution orchestration
Environment x user x request combinatorial execution with progress reporting, streaming sinks, and P50/P95/P99 percentile statistics.

✅ JSON serialization choice
System.Text.Json by default with optional Newtonsoft.Json support via UseNewtonsoftJson flag.

✅ Advanced utilities
Fire-and-forget execution, CURL command logging with sensitive header sanitization, streaming for large payloads, and concurrent request processing.

Quick Example

// Register services with optional features
builder.Services.AddHttpClientUtility(options => {
    options.EnableCaching = true;
    options.EnableResilience = true;
    options.EnableTelemetry = true;
    options.EnableBatchExecution = true;
});

// Or use convenience methods
builder.Services.AddHttpClientUtilityWithCaching();
builder.Services.AddHttpClientUtilityWithResilience(maxRetries: 3, retryDelaySeconds: 1.0);
builder.Services.AddHttpClientUtilityWithAllFeatures();

// Make HTTP requests with automatic retries and caching
var request = new HttpRequestResult<WeatherData>
{
    RequestPath = "https://api.example.com/weather",
    RequestMethod = HttpMethod.Get,
    CacheDurationMinutes = 10, // Optional caching
    AuthenticationProvider = new BearerTokenAuthenticationProvider("your-token")
};

var result = await _httpService.HttpSendRequestResultAsync(request);

if (result.IsSuccessStatusCode)
{
    var weather = result.ResponseResults;
    Console.WriteLine($"Temperature: {weather.Temperature}°C");
}

What's Included

The library provides a complete HTTP client wrapper with:

  • Core Services: IHttpRequestResultService for making HTTP requests
  • Client Service: IHttpClientService typed method-based alternative (Get, Post, Put, Delete)
  • Caching Layer: Automatic response caching with configurable expiration
  • Resilience Layer: Polly-based retry and circuit breaker policies
  • Telemetry Layer: OpenTelemetry integration with metrics and tracing
  • Web Crawler: Full-featured web crawling with robots.txt support (separate package)
  • Batch Execution: Environment x user x request orchestration with progress and P50/P95/P99 percentiles
  • Authentication: Multiple providers (Bearer with dynamic refresh, Basic, API Key) with validation and refresh support
  • JSON Serialization: System.Text.Json default with optional Newtonsoft.Json
  • Fire-and-Forget: Background task execution with structured logging
  • Streaming: Automatic streaming for large responses (10 MB+ threshold)
  • CURL Logging: Request capture as CURL commands with batch processing and sensitive header sanitization
  • Concurrent Requests: Parallel HTTP request execution with semaphore-based throttling
  • Testing Utilities: Mock service for unit testing

Architecture

The library uses a decorator pattern for composing features:

  1. Base Service: Core HTTP request functionality
  2. Cache Decorator: Adds caching capabilities (optional)
  3. Resilience Decorator: Adds Polly policies (optional)
  4. Telemetry Decorator: Adds metrics and tracing (optional)

This design allows you to enable only the features you need while maintaining clean, testable code.
